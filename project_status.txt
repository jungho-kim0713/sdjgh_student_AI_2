컨테이너: Docker + Docker Compose (App + DB + Redis + Celery Worker 통합 관리)

백엔드: Flask (Python 3.12)

데이터베이스: PostgreSQL 15 + pgvector (벡터 검색 지원) (Docker 컨테이너: ai_platform_db)

백그라운드 워커: Celery (문서 벡터화 처리) (Docker 컨테이너: ai_platform_worker)

파일 저장소: 서버 로컬 볼륨 마운트 (./static/uploads, ./static/uploads/knowledge)

외부 접속 포트: 8081 (Docker 내부 8080 매핑)

[ 1.2 최신 구현 사항 (Season 7 - 완료) ]

A. RAG(Retrieval-Augmented Generation) 시스템 통합 및 고도화 - Season 7 (완료)

목표: 교사가 페르소나별로 교육 자료를 업로드하고, 해당 자료를 바탕으로 답변하는 AI 튜터 시스템 구축 및 Contextual Retrieval(스마트 인덱싱) 고도화.

주요 기능:
  - 동적 페르소나 관리: 관리자가 웹 UI에서 페르소나를 추가/수정/삭제 가능
  - 교사 권한 위임: 특정 페르소나의 관리 권한을 특정 교사에게 부여
  - 시스템 프롬프트 편집: 교사가 웹에서 페르소나의 프롬프트를 편집 가능
  - 페르소나별 RAG: 각 페르소나마다 독립적인 지식 베이스 구축 및 활용
  - 스마트 인덱싱 (Contextual Retrieval): 청크 자체만 임베딩하지 않고, Gemini/GPT 모델을 통해 청크의 '핵심 문맥(Context)'을 요약 생성하여 함께 임베딩함으로써 검색 정확도 극대화.

기술 스택:
  - 벡터 DB: pgvector (PostgreSQL 확장)
  - 임베딩 모델: OpenAI text-embedding-3-small (1536차원)
  - 문맥 요약 모델: Gemini 2.5 Flash / GPT-4o-mini
  - 백그라운드 작업: Celery + Redis
  - 청킹 전략: paragraph/sentence/fixed (선택 가능)

완료 사항 (2026.03):
  ✅ Docker Compose 설정 (pgvector, Redis, Celery Worker 통신 연동)
  ✅ RAG 데이터베이스 스키마 완비 (KnowledgeDocument, DocumentChunk 등)
  ✅ 서비스 레이어 완비 (embedding_service.py, chunking_service.py, rag_service.py)
  ✅ Celery 작업 시스템 연동 및 백그라운드 문서 벡터화 자동화
  ✅ OpenAI API, Anthropic, Gemini 등 통합 Provider 연동
  ✅ (최신) RAG 스마트 인덱싱 도입: 문서 청크 분할 시 자동으로 문맥 요약본(Contextual Summary)을 생성하고 메타데이터로 저장하여 검색 품질 혁신적 향상.

[ 1.3 이전 구현 사항 (Season 5.5 - 완료) ]
A. 보이스 인코더 (STT) 고도화 - Season 5.5

데스크탑/모바일 호환성 강화: setRangeText 및 이벤트 트리거 최적화로 모든 기기에서 입력 오류 해결.

중복 입력 방지: isFinal 속성을 활용하여 확정된 문장만 깔끔하게 입력되도록 로직 개선.

실시간 피드백 UI: 입력창 상단에 '실시간 자막 오버레이'를 구현하여, 말하는 내용을 즉각적으로 확인 가능하도록 UX 혁신.

단축키 지원: Ctrl + M 단축키로 마우스 없이도 간편하게 음성 입력 제어 가능.

B. 스마트 복사 버튼 (Smart Copy Button) - Season 5.5

UX 개선: 메시지 박스에 마우스를 올렸을 때만(Hover) 나타나는 세련된 복사 아이콘 구현.

Sticky Position: 긴 메시지를 읽을 때 스크롤을 내려도 복사 버튼이 화면 상단에 '착!' 붙어서 따라오도록 position: sticky 적용.

위치 최적화: 사용자 메시지는 '좌측 상단', AI 메시지는 '우측 상단' 바깥쪽 여백에 배치하여 가독성 확보.

모바일 대응: 모바일 환경에서는 터치 접근성을 위해 항상 표시되도록 반응형 처리.

[ 2.05 Season 6.5. AI 화가 (Imagen) 오류 수정 및 정상화 (완료) ]
기간: 2026.01.24
목표: 'AI 화가' 기능에서 이미지가 생성되지 않고 깨지는(0byte/620byte) 오류 해결.
주요 성과:

원인 규명: Google GenAI SDK가 이미지를 decode된 bytes 형태로 반환함에도, 코드에서 중복 base64 decode를 수행하여 데이터 손상 발생.

코드 수정: routes/chat.py에서 불필요한 base64.b64decode 제거.

검증 완료: Gemini 3.0 Pro Image 모델 등에서 정상적인 이미지 생성 확인 (verify_fix.py).

배포 완료: GitHub Push 및 OCI 서버 배포(Pull & Restart) 완료.

[ 5.5 트러블슈팅 및 버그 수정 (2026-03-03) ]
- **Grok (xAI) 모델 통합**: `app.py` 스키마 초기화 스크립트에 `model_xai`, `restrict_xai` 컬럼 추가.
- **PostgreSQL DB 초기화 오류 해결**: `ALTER TABLE`문에서 SQLite에는 통용되던 `BOOLEAN DEFAULT 0` 문법이 PostgreSQL 15에서 파싱 에러(DatatypeMismatch)를 일으켜 `model_xai` 컬럼이 생성되지 않던 치명적 결함을 발견하여, 명시적인 `BOOLEAN DEFAULT FALSE` 및 `TRUE`로 전체 수정 완료.
- **관리자 권한 버그 해결**: `routes/chat.py` 및 `routes/admin_persona.py` 내의 모든 `user.is_admin` 로직에 명시적 `role == 'admin'` 및 `username == 'admin'` 검증 로직 추가. 이로 인해 모든 페르소나가 누락 없이 다시 출력되도록 수정.
- **UI/UX 수정**: 관리자 패널의 '승인' 열 두 줄 바꿈(Wrapping) 현상 제거 (최소 너비 80px 설정), 모바일 환경에서의 `서대전여고 수업용 AI` 헤더 텍스트 축소 최적화, 모델 드롭다운 병렬 배치.

[ 5.6 성능 및 병목 현상 개선 (2026-03-04) ]
- **동기식 API 호출 블로킹 해결**: `ai_core.py`에 `generate_ai_response_stream`을 추가하여 OpenAI, Anthropic, Google, xAI 모두 Server-Sent Events (SSE) 기반의 비동기 텍스트 스트리밍을 지원하도록 변경.
- **클라이언트 스트리밍 UI 렌더링**: `static/js/modules/chat.js`의 `sendMessageToServer`를 재적용하여 `fetch`의 ReadableStream을 통해 실시간 타자 효과(렌더링) 지원. 다수 동시 접속 시 Gunicorn 워커 스레드 고갈 방지.
- **데이터베이스 인덱스(DB Index) 도입**: `models.py` 내의 잦은 검색 대상 외래키(Foreign Key) 속성인 `session_id`, `user_id`, `role_key`, `persona_id`, `document_id` 등에 `index=True` 부여. 로컬(SQLite) 및 마이그레이션 적용.
- **N+1 및 중복 쿼리 최적화**: `routes/chat.py` 파일 내 파일 배열 검사 반복문을 `ChatFile.id.in_(file_ids)`로 교체 및 무수히 잦던 DB `commit()`을 `flush()`로 변경하여 트랜잭션 오버헤드를 감소.
